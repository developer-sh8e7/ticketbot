# Opus Solutions

منصة بيع بوتات ديسكورد كاشتراكات (SaaS). ثلاثة منتجات بوت مستقلة فوق نواة مشتركة، تديرها طبقة تحكم واحدة تتولّى بركة التوكنات، التفعيل التلقائي بعد الدفع، ودورة حياة الاشتراك.

> **اقرأ أولاً:** [`ARCHITECTURE.md`](./ARCHITECTURE.md) للتصميم الكامل، و [`MIGRATION_MAP.md`](./MIGRATION_MAP.md) لمعرفة أين يذهب كل ملف من المشروع القديم.

## المنتجات الثلاثة (مفصولة)

| الحزمة | المنتج | المصنع المُصدَّر |
|--------|--------|------------------|
| `apps/ticket-bot` | بوت التكتات (تذاكر، لجان، تصعيد، تفريغ) | `createTicketBot` |
| `apps/voice-rooms-bot` | بوت الغرف الصوتية المؤقتة | `createVoiceRoomsBot` |
| `apps/general-bot` | البوت العام (إدارة، مستويات، اقتصاد، ألعاب) | `createGeneralBot` |
| `apps/orchestrator` | طبقة التحكم (التوكنات + التفعيل + المدير + webhook) | — |
| `packages/core` | النواة المشتركة | — |

## البدء

```bash
# 1) المتطلبات: Node 20+, pnpm 9+
corepack enable

# 2) التبعيات
pnpm install

# 3) البيئة
cp .env.example .env   # املأ Supabase + TOKEN_ENCRYPTION_KEY + ...

# 4) قاعدة البيانات (انظر db/README.md)
#    طبّق ملفات db/schema/*.sql بالترتيب على Supabase

# 5) استورد إعدادات السيرفر الخاص (اختياري)
pnpm tsx db/seed/import_server_configs.ts

# 6) البناء والتشغيل
pnpm build
pnpm start:orchestrator        # يشغّل طبقة التحكم + webhook + يدير كل البوتات

# تشغيل بوت مفرد للتطوير:
pnpm dev:ticket                # يتطلب TICKET_DEV_TOKEN و TICKET_DEV_GUILD
```

## كيف يعمل الشراء

1. الزبون يسجّل دخوله بـ Discord OAuth في الموقع → يُنشأ سجل في `accounts` (**الربط الإجباري**).
2. يدفع عبر PayPal → الموقع يستدعي `POST /internal/provision` في الأوركستريتر.
3. `provision_instance()` ذرّياً: يتحقق من الربط → يسحب توكناً من البركة (`claim_token`) → ينشئ نسخة بوت `active` → يسجّل الاشتراك.
4. مدير النسخ يفك تشفير التوكن، يجلب إعدادات السيرفر، ويشغّل البوت — يدخل سيرفر الزبون فوراً.
5. عند الانتهاء: يتوقف البوت، **والإعدادات تبقى محفوظة**. عند التجديد يرجع نفس البوت بإعداداته.
