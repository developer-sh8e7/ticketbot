# UI Need Summary

## User Request
إعادة بناء قسم باقات Opus Solutions على هيئة مسار بطاقات WebGL مستوحى من قسم الوكلاء في HUMAIN ONE، مع اسم الباقة والسعر داخل كل بطاقة. التفاصيل لا تظهر عند hover؛ يبدأ التسلسل فقط بضغطة الماوس اليسار او tap: ترتفع البطاقة، تنطلق اشارة مستقيمة منها الى الاعلى، تنعطف الى لوحة ثابتة اعلى اليسار، ثم تتكون المعلومات بانيميشن كامل متعدد المراحل. ممنوع فتح شاشة كاملة.

## Product Goal
جعل استعراض باقات المواقع والتطبيقات تجربة بصرية مميزة وسهلة الفهم، مع إبقاء تفاصيل الباقة في سياق الصفحة وقيادة المستخدم إلى طلب المشروع.

## Target Users
طلاب، أصحاب أفكار، وأعمال تبحث عن مواقع أو تطبيقات أو أنظمة رقمية.

## Primary Scenarios
- مشاهدة الباقات وأسعارها أثناء الحركة المستمرة.
- hover يمنح البطاقة استجابة خفيفة فقط بدون خط او معلومات.
- ضغطة يسار او tap تطلق تسلسل البطاقة والاشارة واللوحة.
- سحب المسار أفقيا على اللمس أو سطح المكتب.
- استخدام بديل نصي عند غياب WebGL أو تفضيل تقليل الحركة.

## Platform and Surface
Web responsive داخل Next.js 16 / React 19، صفحة `/packages`، باستخدام Three.js المثبت في المشروع.

## Desired Quality Level
- Fast / Standard / Deep: Deep
- MVP or production-ready: Production-ready

## Constraints
- Tech/framework: Next.js, React, TypeScript, Three.js, Tailwind/CSS.
- Brand/design system: ثلجي أبيض مع أخضر ليموني ومينت وتيل، مطابق لاتجاه Opus الحالي ومتأثر بـ HUMAIN.
- Accessibility: تفاصيل DOM دلالية، بديل WebGL، ودعم `prefers-reduced-motion`.
- Performance: 64 نسخة بطاقة مع مشاركة الهندسة، حد لنسبة البكسل، وإيقاف الرسم خارج مجال الرؤية.

## Decisions Confirmed With User
- الوضع العميق والتنفيذ بدون اسئلة اضافية.
- الحفاظ على باقات Opus الحالية وأسعارها.
- التفاصيل بالضغط فقط وليس hover.
- اللوحة اعلى اليسار دائما.
- الاشارة مستقيمة صاعدة ثم انعطاف مرتب، بروح طاقة رقمية وليست كهرباء حرفية.
- عدم فتح Modal أو تجربة ملء الشاشة.
