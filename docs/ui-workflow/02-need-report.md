# UI Requirements Report

## Product Positioning

قسم الباقات هو نقطة تحويل من الاستكشاف الى طلب مشروع. يجب ان يبدو كعرض تقني مميز من Opus لا كشبكة اسعار SaaS تقليدية، مع بقاء السعر والخطوة التالية واضحين.

## Target Users / Personas

1. طالب يريد مشروع تخرج ويقارن السعر بسرعة.
2. صاحب فكرة يحتاج صفحة او تطبيق ولا يعرف التفاصيل التقنية.
3. نشاط تجاري يريد متجر او موقع تعريفي ويبحث عن جهة احترافية.
4. مستخدم جوال يعتمد اللمس ولا يملك hover.

## Jobs To Be Done

- اعرف انواع المشاريع التي تقدمها Opus وسعر البداية لكل نوع.
- اركز على باقة واحدة بدون فقدان سياق باقي الخيارات.
- اشوف اهم ما تشمله الباقة ثم انتقل لطلب المشروع.
- اسحب المسار او اتركه يتحرك بدون تعلم واجهة جديدة.

## Key User Scenarios

- Desktop: hover يرفع البطاقة ويعرض التفاصيل، click يثبتها.
- Mobile: tap يثبت البطاقة واللوحة، وزر الاغلاق يعيد المسار.
- Filter: اختيار التصنيف يخفض ظهور البطاقات الاخرى.
- Reduced motion: يتوقف التحرك التلقائي وتبقى الاستجابة الوظيفية.
- WebGL failure: يظهر مسار HTML افقي قابل للسحب.

## Feature and Page Priority

| Priority | Feature/Page | User Value | Notes |
|---|---|---|---|
| P0 | مسار Three.js في `/packages` | تمييز العرض وفهم الخيارات | 64 بطاقة وحركة مستمرة |
| P0 | اسم + سعر داخل البطاقة | مقارنة فورية | محتوى مختصر عالي التباين |
| P0 | لوحة تفاصيل غير كاملة الشاشة | بقاء السياق | يمين/يسار تلقائي |
| P0 | تفاعل hover/tap | دعم كل الاجهزة | click يثبت الحالة |
| P1 | فلاتر التصنيف | تقليل الخيارات | تخفيض opacity دون حذف المسار |
| P1 | fallback وreduced motion | وصول وثبات | بدون منع المحتوى |
| P2 | قياس تحليلات التفاعل | تحسين التحويل | خارج نطاق هذا التعديل |

## Required States

- Empty: غير متوقع لان الباقات ثابتة؛ لا يتم تشغيل المشهد عند مصفوفة فارغة.
- Loading: بطاقة skeleton ثلجية حتى تجهز الخطوط والخامات.
- Error: مسار HTML افقي عند فشل WebGL او فقدان السياق.
- Success: بطاقة مرفوعة، خط ظاهر، لوحة معلومات وCTA.
- Edge cases: سحب لا يعد click، خروج المؤشر يرجع البطاقة، اغلاق اللوحة يلغي التثبيت، تغيير العرض يعيد حساب الكاميرا والقياسات.

## Success Metrics

- لا اخطاء TypeScript او build.
- لا Modal او scroll lock عند اختيار الباقة.
- عمل hover وclick وtouch والسحب.
- عدم تجاوز لوحة الجوال عرض الشاشة.
- توقف الحركة الذاتية مع reduced motion.

## Assumptions / Risks

- الاسعار الحالية هي اسعار بداية صحيحة.
- 64 جسم RoundedBox مقبول على الاجهزة الحديثة؛ يتم رسم المرئي فقط وحد pixel ratio.
- النص داخل Canvas ليس دلاليا، لذلك تبقى قائمة HTML نصية موازية.
- التطابق المقصود سلوكي وبصري وليس نسخ كود او هوية HUMAIN.
